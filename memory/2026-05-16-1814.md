assistant: ## 数据记录流程

### 1. 原始数据写入

当你说一句话，比如：

> "买电影票花了200"

脚本会：
1. **识别金额**：¥200
2. **识别用途**："电影票" → 自动分类为 `娱乐`
3. **记录到 `transactions` 表**：
   ```
   id | type    | amount | category | description | date
   ---|----------|--------|----------|-------------|----------
   1  | expense  | 200    | 娱乐      | 电影票       | 2026-05-16
   ```
4. **更新 `balance` 表**：余额 -200

---

### 2. 汇总数据是怎么自动生成的？

每次记录一笔收支，**同时**会更新周/月/年三张汇总表：

```
record_expense(200, "电影票", "2026-05-16")
    │
    ├── ① 更新 weekly_summary（第20周）
    │       year=2026, week=20
    │       total_expense += 200
    │       category_expenses["娱乐"] += 200
    │
    ├── ② 更新 monthly_summary（2026年5月）
    │       year=2026, month=5
    │       total_expense += 200
    │       category_expenses["娱乐"] += 200
    │
    └── ③ 更新 yearly_summary（2026年）
            year=2026
            total_expense += 200
            category_expenses["娱乐"] += 200
```

---

### 3. 三张汇总表结构

| 表 | 主键 | 字段 |
|---|---|---|
| `weekly_summary` | year + week | total_income, total_expense, **category_expenses (JSON)** |
| `monthly_summary` | year + month | total_income, total_expense, **category_expenses (JSON)** |
| `yearly_summary` | year | total_income, total_expense, **category_expenses (JSON)** |

`category_expenses` 存的是 JSON，比如：
```json
{"饮食": 500, "娱乐": 200, "交通": 80}
```

---

### 4. 查询时怎么用？

你说"本月支出多少" → 查询 `monthly_summary`
你说"年度总结" → 查询 `yearly_summary`

不需要每次从原始明细重新聚合，汇总表里直接有：
- 总收入 / 总支出 / 结余
- 各类别支出金额
- 占比可以直接算出来

---

### 5. 数据流向图

```
你说"买电影票花了200"
         │
         ▼
   ┌─────────────┐
   │ transactions │ ← 明细表（每笔都记）
   └─────────────┘
         │
         ▼
┌────────────────────────────┐
│   汇总表（三张表自动更新）    │
├────────────────────────────┤
│ weekly_summary  ← 周维度     │
│ monthly_summary ← 月维度     │
│ yearly_summary  ← 年维度     │
└────────────────────────────┘
         │
         ▼
 你查"本月支出" → 读 monthly_summary → 直接出结果
```

这样设计的好处是：**写入会慢一点**（要更新4张表），但**查询非常快**，不管查周报月报年报都是秒出，不用重新聚合原始数据。
