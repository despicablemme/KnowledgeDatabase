## 架构概览

```
第二个 QQ Bot (新 AppID)
    ↓
binding 路由规则
    ↓
新 Agent (独立 workspace + SOUL.md + memory)
```

每个 agent 有自己独立的：
- `workspace`（文件、人格设定、工作区）
- `SOUL.md` / `IDENTITY.md` / `USER.md`
- Session 历史和记忆

## 你需要做的

### 第一步：创建第二个 QQ Bot

去 [QQ 开放平台](https://q.qq.com/) 创建第二个机器人，获得新的 **AppID** 和 **AppSecret**。

### 第二步：创建新 agent 工作区

```bash
openclaw agents add alt
```

这会在 `~/.openclaw/workspace-alt` 创建新人格的工作区，并创建对应的 `SOUL.md`、`AGENTS.md` 等文件。

### 第三步：配置第二个 QQ Bot 账号 + 绑定

假设第二个 bot 的 AppID 是 `222222222`，在 `openclaw.json` 里加：

```json5
{
  channels: {
    qqbot: {
      appId: "第一个机器人的AppID",       // 你现有的
      clientSecret: "第一个机器人的Secret",
      accounts: {
        alt: {                            // 第二个 bot 账号
          enabled: true,
          appId: "222222222",
          clientSecret: "第二个机器人的Secret"
        }
      }
    }
  },
  agents: {
    list: [
      { id: "main", default: true },      // 你现在的人格
      { id: "alt" }                        // 新人格
    ]
  },
  bindings: [
    { agentId: "main", match: { channel: "qqbot" } },           // 默认路由
    { agentId: "alt", match: { channel: "qqbot", accountId: "alt" } }
  ]
}
```

`bindings` 的匹配规则是**最具体优先**，所以来自第二个 bot（`accountId: "alt"`）的消息会路由到 `alt` agent，来自第一个 bot 的消息路由到 `main` agent。

### 第四步：给新人格写灵魂

编辑 `~/.openclaw/workspace-alt/SOUL.md`，给第二个人格定义独立的性格、语气、能力偏好。

### 第五步：重启

```bash
openclaw gateway restart
```
