# DailyLearn - 每日学习笔记

## 核心规则

1. **文件夹**：`~/Documents/KnowLedgeDatabase/DailyLearn/`
2. **文件名**：`{日期}-{主题}.md`，如 `2026-04-27-Python装饰器.md`
3. **同一天同一主题不重复创建**——检测到今日已存在该主题文件时，追加内容而非新建
4. **主题不明确时询问用户**
5. **记录原文链接**——收到链接类内容时，必须在文档中记录原始链接地址
6. **内容要详尽无遗漏**——总结时不要概括，要详尽、全面、无重复地写入文档

## 工作流程

### 收到"今日daily learn"笔记时：

1. 解析日期（今天）
2. 判断主题：
   - 用户明确给出主题 → 直接使用
   - 用户未说明主题 → 先询问主题
3. 检查今日是否已有该主题文件：
   - 检查 `{folder}/{日期}-{主题}.md` 是否存在
   - **存在**：读取现有内容，追加新内容（用分隔线隔开）
   - **不存在**：创建新文件，写入内容
4. 回复用户告知保存位置

### 收到链接时的处理：

1. **记录原文链接**：在文档最上方或固定位置记录原始 URL
2. **获取内容**：尝试抓取链接内容
3. **详尽总结**：
   - 不要概括，要详尽记录原文的每一个要点
   - 没有重复内容，同一内容只写一次
   - 保持原文结构，条理清晰
4. 如果无法访问链接（如 localhost），询问用户提供具体内容

### 文件格式

```markdown
# {主题}

> 原文链接：{URL}

{内容}

---
添加时间：{完整时间戳}
```

## 状态追踪

为避免同一天重复创建文件，使用 `~/.openclaw/workspace/memory/daily-learn-state.json` 追踪今日状态：

```json
{
  "lastDate": "2026-04-27",
  "topicsCreatedToday": ["Python装饰器", "Git进阶"]
}
```

每次创建或追加文件后更新此文件。每次处理前先检查 `lastDate` 是否为今天，不是则重置。

## 示例

**用户说**："今天学了 Python 装饰器，这是我的笔记：装饰器是..."

**操作**：
1. 日期：2026-04-27
2. 主题：Python装饰器
3. 检查文件：`DailyLearn/2026-04-27-Python装饰器.md`
4. 不存在 → 创建并写入

**回复**：已保存到 `~/Documents/KnowLedgeDatabase/DailyLearn/2026-04-27-Python装饰器.md`

---

**用户发送链接**："daily learn 这个链接"

**操作**：
1. 日期：2026-04-28
2. 主题：待确认（先询问用户或从内容推断）
3. 记录原文链接
4. 抓取内容，详尽写入文档
5. 回复保存位置

**回复**：已保存到 `~/Documents/KnowLedgeDatabase/DailyLearn/2026-04-28-{主题}.md`，包含原文链接。
